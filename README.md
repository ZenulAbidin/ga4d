# ga4d

`ga4d` is a C99 OpenGL/GLFW viewer for the 4D geometry notes in `notes/ai-agents/4D Geometry Through Clifford Algebra.md`.

Current version: `1.1.0`.

The program focuses on Clifford/geometric algebra:

- 4D points are grade-1 vectors.
- Rotations are unit rotors generated by bivectors.
- Vector rotation is computed with the sandwich product `R v reverse(R)`.
- The UI can edit either coordinate bivectors `E12..E34` or the self-dual/anti-self-dual split `L1..L3` and `R1..R3` from the notes.

Version `1.1.0` adds icon extrusion examples:

- 2D raster-like icon silhouettes swept through the `w` axis.
- External PNG icons loaded with `--icon file.png` and converted into bounded 4D wire extrusions.
- A simple 3D house icon swept through `w`, showing the 4D analogue of extruding a 3D object.

The earlier published build is tagged `v1.0.0`; this icon-extrusion build is tagged `v1.1.0`.

## Build Dependencies

Required tools and libraries:

- A C99 compiler such as `cc`, GCC, or Clang.
- GNU Make.
- CMake, used only when the Makefile has to build GLFW from source.
- Git, used only when the Makefile has to clone GLFW from source.
- OpenGL development headers and libraries.
- X11 development headers and libraries: X11, Xrandr, Xi, Xxf86vm, Xcursor, and Xinerama.

Optional but useful:

- `pkg-config`, so the Makefile can find a system GLFW and optional Wayland/EGL libraries.
- System GLFW 3, for example `libglfw3-dev`. If this is missing, `make` clones GLFW 3.4 from GitHub into `vendor/glfw` and builds it locally.
- Wayland/EGL development packages plus `weston`, only needed for `make wayland-smoke` and for Wayland support in the vendored GLFW build.
- No libpng package is needed. PNG decoding uses the committed single-header `vendor/stb/stb_image.h`.

On Debian or Ubuntu, the base dependencies are typically:

```sh
sudo apt install build-essential make cmake git pkg-config libgl1-mesa-dev libx11-dev libxrandr-dev libxi-dev libxxf86vm-dev libxcursor-dev libxinerama-dev
```

For optional system GLFW and headless Wayland smoke testing:

```sh
sudo apt install libglfw3-dev libwayland-dev wayland-protocols libxkbcommon-dev libegl1-mesa-dev weston
```

If system GLFW is not installed and `vendor/glfw` is absent, the first `make` needs network access so it can clone `https://github.com/glfw/glfw.git`.

## Build

```sh
make
make check
```

The `Makefile` first tries `pkg-config glfw3`. If GLFW is not installed, it builds the vendored GLFW source in `vendor/glfw` with CMake and links it statically.
For vendored GLFW 3.4, the build applies `patches/glfw-3.4-wayland-null-seat.patch` so render-only headless Wayland smoke tests do not crash when the compositor has no input seat.

Useful targets:

- `make run` launches the viewer.
- `make check` runs no-window self-tests for topology, Clifford rotor length preservation, self/anti-dual conversion, projection sanity, and a visual-geometry sweep across presets/bases/focal distances/slices.
- `make smoke` renders a few frames, reads framebuffer pixels, and fails if the frame looks blank or OpenGL reports an error. It needs a working X/desktop display.
- `make headless-smoke` tries the same pixel-checked render smoke through GLFW's null platform and EGL. It still needs an EGL stack that can create an OpenGL context.
- `make wayland-smoke` runs the same pixel-checked smoke against a Wayland compositor at `/tmp/ga4d-runtime/ga4d-wayland`. One way to provide that compositor is:

```sh
mkdir -p /tmp/ga4d-runtime
chmod 700 /tmp/ga4d-runtime
env XDG_RUNTIME_DIR=/tmp/ga4d-runtime weston --backend=headless-backend.so --socket=ga4d-wayland --width=1280 --height=800 --idle-time=0 --no-config
```

Then in another shell:

```sh
make wayland-smoke
```

- `make clean` removes local build output.

## PNG Icons

Load a PNG icon and start directly on its 4D extrusion:

```sh
./bin/ga4d --icon path/to/icon.png
```

Check PNG decoding and mesh generation without opening an OpenGL window:

```sh
./bin/ga4d --probe-icon path/to/icon.png
```

Transparent PNGs use alpha as the silhouette. PNGs without transparency treat non-white pixels as the icon. Large images are downsampled to a bounded grid before extrusion so the viewer remains responsive.

## Controls

- Mouse left drag: orbit camera.
- Mouse right drag: pan.
- Mouse wheel: zoom.
- `Tab`: cycle object: tesseract, 16-cell, 5-cell, hypersphere.
- `M`: cycle view: projection, slice, both.
- `R`: toggle basis: coordinate bivectors or self/anti-dual split.
- `1`..`6`: select the active generator in the current basis.
- Left/right arrows: change selected generator angle.
- Up/down arrows: change selected generator speed.
- `0`: zero selected generator.
- `[` and `]`: adjust 4D focal distance.
- `,` and `.`: move the slice along `w`.
- `W/A/S/D`: keyboard camera orbit.
- `Z/X`: keyboard zoom.
- `Space`: pause animation.
- `G`: toggle grid.
- `V`: toggle vertex points.
- `C`: toggle `w` depth color cue.
- `F1`..`F8`: presets.
- `F6`: 2D arrow icon extruded through `w`.
- `F7`: PNG icon extrusion. Without `--icon`, this shows a built-in ring sample.
- `F8`: 3D house icon extruded through `w`.
- `Esc` or `Q`: quit.

## Notes

This is intentionally plain C99. There are no C++ templates, classes, inheritance, GLM dependency, or shader pipeline requirement. The renderer uses conservative fixed-function OpenGL through GLFW to reduce driver/setup fragility.
